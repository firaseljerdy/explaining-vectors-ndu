SECTION 01

Why Neural Networks?

Some problems are trivially easy for computers but hard for humans. Ask a computer to find the square root of 964,324 and it spits out 982 in nanoseconds. Ask me, and I'll be there for a while.

But flip the problem around. Show a toddler a picture of a kitten or a puppy and they'll classify it instantly. Getting a computer to do that reliably took decades of research. Pattern recognition, language understanding, decision-making in uncertain environments—these tasks demand a different kind of computation.

Neural networks are a family of algorithms inspired by the brain. Not an accurate model of the brain—more like a cartoon of it. But that cartoon turns out to be extraordinarily useful. It can learn from examples, generalize to unseen data, and adapt its internal structure over time.

My Takeaway

Throughout this course we've written the rules ourselves: F = ma, friction = −μ·v̂, steer toward target. With neural networks, the paradigm flips. Instead of writing rules, we give the system examples and let it discover the rules. That's the fundamental shift.

SECTION 02

The Biological Neuron

A biological neuron has three key parts:

  • Dendrites — branch-like extensions that receive electrical signals from other neurons.
  • Soma (cell body) — processes the incoming signals and decides whether the neuron should fire.
  • Axon — a long projection that transmits the output signal to the next neuron.

If the combined strength of incoming signals exceeds a threshold, the neuron fires—it sends its own electrical pulse down the axon to downstream neurons. If the signals are too weak, nothing happens. This all-or-nothing behavior is the basis of the activation function in artificial neurons.

The Artificial Version

An artificial neuron does the same thing in code: receive inputs (dendrites), compute a weighted sum (soma), pass it through an activation function (fire or don't fire), and produce an output (axon). Every neural network, from the simplest perceptron to massive language models, is built by connecting many of these artificial neurons together.

Of course, real brains are unfathomably more complex. Each of your 100 billion neurons connects to roughly 10,000 others, creating a network of a quadrillion connections. We're going to work with one neuron, three inputs, and a handful of weights. But the principle is the same.

SECTION 03

The Perceptron

The perceptron is the simplest neural network: a single artificial neuron. Invented in 1957 by Frank Rosenblatt at the Cornell Aeronautical Laboratory, it takes multiple inputs, multiplies each by a weight, sums them, and passes the result through an activation function to produce a single output.

The Algorithm in Three Steps

  1. Weight the inputs: Multiply each input by its corresponding weight.
  2. Sum: Add all the weighted inputs together.
  3. Activate: Pass the sum through the sign function. Positive → +1, Negative → −1.
output = sign( x₁·w₁ + x₂·w₂ + 1·wbias )
The perceptron's complete feedforward formula.

The Bias Input

Notice the third input—the bias. Its value is always 1, but it has its own weight. Why? Without it, the point (0, 0) would always produce a sum of 0 regardless of the weights (anything times zero is zero). The bias gives the perceptron the ability to shift its decision boundary away from the origin.

The Click Moment

Think of the bias as the y-intercept of a line. Without it, every decision boundary must pass through the origin. The bias weight lets the line slide up or down, giving the perceptron far more flexibility.

SECTION 04

Feedforward: Making a Guess

The feedforward method is how the perceptron makes a prediction. Data flows in one direction—forward—from inputs through the weighted sum to the output. There's no looping, no feedback. Just a straight pipeline.

feedforward(inputs) {
  let sum = 0;
  for (let i = 0; i < this.weights.length; i++) {
    sum += inputs[i] * this.weights[i];
  }
  return this.activate(sum);
}

Let's trace through an example. Say the inputs are [0.7, -0.3, 1] (two features plus the bias) and the weights are [0.6, -0.4, 0.3]:

  • 0.7 × 0.6 = 0.42
  • −0.3 × −0.4 = 0.12
  • 1.0 × 0.3 = 0.30
  • Sum = 0.84
  • sign(0.84) = +1 (fire!)

At this point the perceptron has no idea if +1 is the right answer. The weights started random, so it's essentially guessing. But that's okay—next comes training.

Common Mistake

Forgetting the bias input. If your inputs array has 2 values but your perceptron expects 3, the feedforward loop will skip the bias weight entirely. Always include the 1 at the end: [x, y, 1].

SECTION 05

The Sign Activation Function

The activation function is what makes a neuron more than just arithmetic. Without it, the perceptron would just output a continuous number. The activation introduces a nonlinear decision—it forces the output into one of two categories.

activate(sum) {
  if (sum >= 0) {
    return 1;    // fire
  } else {
    return -1;   // don't fire
  }
}

The sign function is a step function: it's −1 everywhere below 0 and +1 everywhere at or above 0. There's no in-between—it's binary. This is perfect for classification: is this student admitted or rejected? Is this email spam or not?

Other Activation Functions

In deeper neural networks, smoother activation functions like sigmoid (S-shaped curve between 0 and 1) or ReLU (0 for negatives, linear for positives) are used because they allow gradients to flow through for backpropagation. But for a single perceptron doing binary classification, the sign function is sufficient and easy to reason about.

Biological Parallel

A biological neuron fires or doesn't fire—it's essentially a sign function operating on the combined strength of incoming signals. If the electrochemical potential exceeds a threshold, the neuron sends a pulse. Otherwise, silence. Our activate(sum) is a simplified version of this same mechanism.

SECTION 06

Supervised Learning: Teaching the Perceptron

A perceptron with random weights is useless—it has a 50/50 chance of getting any answer right. To make it useful, we need to train it. The strategy is called supervised learning: we show it examples where we already know the correct answer, and let it learn from its mistakes.

The training loop is simple:

  1. Give the perceptron an input with a known answer (the label).
  2. Ask the perceptron to guess.
  3. Calculate the error: desired − guess.
  4. Adjust the weights based on the error.
  5. Repeat from step 1.
error = desired − guess
The error is the gap between what the answer should be and what the perceptron guessed.

Since both desired and guess are either +1 or −1, there are only three possible error values:

  • 0 — correct! No adjustment needed.
  • +2 — desired was +1, guessed −1. Needs to fire more.
  • −2 — desired was −1, guessed +1. Needs to fire less.
Familiar Pattern

Remember the steering force from Chapter 5? steering = desired velocity − current velocity. That's also an error! The vehicle's current velocity is its "guess" about where to go, and the steering force corrects it. A perceptron's weight adjustment works the same way—nudging the weights in the direction that reduces the error.

SECTION 07

The Weight Update Rule

The weight update is the beating heart of learning. For each weight, we compute how much to change it:

wnew = wold + error × input × learningRate
Each weight is nudged proportionally to the error, the input, and the learning rate.

Why multiply by the input? Because the input determines how much each weight contributed to the error. A large input means that weight had a big influence on the sum, so it deserves a bigger correction. A small input means it barely mattered, so a small correction suffices.

train(inputs, target) {
  let guess = this.feedforward(inputs);
  let error = target - guess;

  for (let i = 0; i < this.weights.length; i++) {
    this.weights[i] += error * inputs[i] * this.lr;
  }
}

The Learning Rate

The learning rate (we use 0.01) controls the size of each adjustment:

  • Too high (e.g. 1.0) — weights swing wildly, overshooting the correct values. The perceptron might never settle.
  • Too low (e.g. 0.0001) — learning is painfully slow. It'll get there eventually, but you might not have the patience.
  • Just right (e.g. 0.01) — steady, reliable convergence.
Same Trade-Off Everywhere

This is the same tension as the maxForce in steering behaviors. High max force → quick turns but jittery. Low max force → smooth but sluggish. The learning rate is the neural network's version of max steering force.

SECTION 08

The Decision Boundary

When a perceptron classifies 2D points, it's essentially drawing a line through the space. Points on one side → +1, points on the other → −1. This line is called the decision boundary.

The equation of the decision boundary comes directly from the weights. The perceptron fires when the weighted sum equals zero:

w₀·x + w₁·y + w₂·1 = 0
Rearranges to: y = −(w₀/w₁)·x − (w₂/w₁)

That's the equation of a line with slope −(w₀/w₁) and y-intercept −(w₂/w₁). As training adjusts the weights, the slope and intercept change—the line rotates and shifts until it correctly separates the two classes.

Linear Separability

A perceptron can only solve problems where a single straight line can separate the two classes. This is called linear separability. Many real-world problems are linearly separable (our admissions classifier is one), but some aren't.

The classic counterexample is XOR (exclusive or). Two inputs can both be true or both be false—no single line can separate XOR's outputs. For that, you need multiple layers of neurons. But for our purposes, one perceptron is plenty.

Limitation to Remember

A single perceptron = one line. If your data can't be split by a line, the perceptron will never converge to 100% accuracy, no matter how long you train. For complex patterns, you'd stack multiple perceptrons into layers—that's how deep learning works. But that's a story for another day.

SECTION 09

Train / Test Split

Imagine a student who memorizes every answer in the textbook but can't solve a single new problem. That's overfitting—the model has memorized the training data instead of learning general patterns.

To detect this, we split the data before training:

  • Training set (80%): The data the perceptron learns from.
  • Testing set (20%): Held back—the perceptron never sees this during training. It's the "final exam."

In our admissions exercise, that's 32 students for training and 8 for testing (4 admitted + 4 rejected in each split).

// 80% Train (16 admitted + 16 rejected = 32)
processData(rawAdmitted.slice(0, 16),  1,  trainingData);
processData(rawRejected.slice(0, 16), -1, trainingData);

// 20% Test (4 admitted + 4 rejected = 8)
processData(rawAdmitted.slice(16, 20),  1,  testingData);
processData(rawRejected.slice(16, 20), -1, testingData);

Two Types of Accuracy

Training accuracy tells you how well the perceptron fits the data it was trained on. Testing accuracy tells you how well it generalizes—can it make correct predictions on data it's never seen?

If training accuracy is 100% but testing accuracy is 50%, the perceptron memorized without learning. If both are high, it truly learned the underlying pattern.

Why This Is Beautiful

The train/test split is one of the most important ideas in all of machine learning, and it's deceptively simple: just don't show the model all your data. Keep some back as a reality check. It's the difference between a student who understands and one who just memorized.

SECTION 10

Class Demo: University Admissions

Here's the full exercise we built in class. The scenario: a university admits or rejects students based on two exam scores. We have 40 students with known outcomes. The perceptron must learn the decision boundary from 32 of them and prove it works on the remaining 8.

The Data

20 admitted students (exam scores in the 55–95 range) and 20 rejected students (scores in the 20–60 range). We normalize the scores from 0–100 into −1 to +1 using map(). Admitted students are labeled +1, rejected students −1.

let rawAdmitted = [
  [85,75], [72,80], [90,65], [60,85], [78,70],
  [88,60], [65,90], [70,75], // ... 20 total
];
let rawRejected = [
  [45,50], [30,60], [50,40], [35,55], [40,45],
  [55,35], [25,65], [48,42], // ... 20 total
];

The Perceptron

class Perceptron {
  constructor(numInputs, learningRate) {
    this.weights = [];
    this.learningRate = learningRate;
    for (let i = 0; i < numInputs; i++) {
      this.weights[i] = random(-1, 1);
    }
  }

  feedforward(inputs) {
    let sum = 0;
    for (let i = 0; i < this.weights.length; i++) {
      sum += inputs[i] * this.weights[i];
    }
    return this.activate(sum);
  }

  activate(sum) {
    if (sum >= 0) return 1;
    else return -1;
  }

  train(inputs, target) {
    let guess = this.feedforward(inputs);
    let error = target - guess;
    for (let i = 0; i < this.weights.length; i++) {
      this.weights[i] += error * inputs[i] * this.learningRate;
    }
  }
}

The Training Loop

Each frame, we feed one training example through the perceptron. After cycling through all 32, we check the training accuracy. When it hits 100%, we stop training and run the test:

function draw() {
  if (!converged) {
    let d = trainingData[trainIndex];
    perceptron.train([d.x, d.y, 1], d.label);
    trainIndex = (trainIndex + 1) % trainingData.length;
    trained++;

    if (trainIndex === 0) {
      let correct = 0;
      for (let d of trainingData)
        if (perceptron.feedforward([d.x, d.y, 1]) === d.label)
          correct++;
      trainAccuracy = correct / trainingData.length;

      if (trainAccuracy >= 1.0) {
        converged = true;
        // Grade the "Final Exam"
        let testCorrect = 0;
        for (let t of testingData)
          if (perceptron.feedforward([t.x, t.y, 1]) === t.label)
            testCorrect++;
        testAccuracy = testCorrect / testingData.length;
      }
    }
  }
}

The Visualization

The background is divided into colored regions based on the perceptron's current predictions—green-tinted for "admit" and red-tinted for "reject." A decision boundary line is drawn using the current weights. Training data appears as circles, testing data as squares. Green outlines mean the perceptron classified that point correctly; red outlines mean it got it wrong.

What's Happening Under the Hood

Watch the decision boundary line rotate and shift as training progresses. Each weight update nudges the line a little. After a few hundred training steps, the line settles into the right position—perfectly separating admitted students from rejected ones. Then the "final exam" runs on the 8 unseen students. If both accuracies are high, the perceptron truly learned the pattern, not just memorized the examples.

Full Pipeline Recap

1. Initialize random weights.
2. Feedforward: compute weighted sum, apply sign activation.
3. Calculate error: desired − guess.
4. Update weights: w += error × input × learning rate.
5. Repeat until converged.
6. Test on unseen data to verify generalization.

This single-neuron classifier is the foundation of everything in neural networks. Stack multiple perceptrons into layers, replace the sign function with a smooth activation, add backpropagation to train the hidden layers, and you've built a deep neural network. But the core loop—feedforward, error, weight update—remains exactly the same.

That wraps Chapter 6. From biological neurons to a working admissions classifier—one perceptron at a time.
